# SaaS Mass-Production Protocol

This repository was generated by ArkNet Forge under the Stitch design-bundle mass-production contract.

## Source of Truth Order

1. strategy.md defines product truth, personas, pains, mechanic, value prop, journey, metrics, and voice.
2. DESIGN.md defines design-system truth: colors, typography, spacing, components, motion, vibe, and accessibility.
3. docs/STITCH_EXPORT_BUNDLE.md defines website-design truth when Stitch screen/code exports are supplied.
4. docs/DESIGN_SYSTEM_SUMMARY.md is the parsed implementation summary.
5. docs/STITCH_REFINEMENT_PROMPTS.md is the prompt path for Stitch iteration.
6. Code is only valid when it respects all files above.

## Non-Negotiable Rules

1. Always read DESIGN.md before writing UI code.
2. Always read docs/STITCH_EXPORT_BUNDLE.md before writing screen code when it is supplied.
3. Always read strategy.md before writing copy.
4. Never ship generic Tailwind/shadcn defaults.
5. Never invent mismatched benefits; all claims must derive from the idea and strategy.
6. Generate one screen at a time, then critique against DESIGN.md, STITCH_EXPORT_BUNDLE.md, and strategy.md.
7. Use Zoom-Out -> Zoom-In Stitch prompts, one-change iterations, variants, and critique loops.
8. The product page must be a real category-specific workflow, not a renamed textarea demo.
9. Mobile layout must be designed first, with readable type, usable touch targets, and no overlapping content.
10. Category visuals must fit the market: aviation operational, barber craft-led and bookable, beverage sensory and commercial, governance auditable, knowledge SaaS verified.
11. Do not use ArkNet dark blue unless the generated product is ArkNet-branded.

- Generic "Try the live workflow" copy.
- Generic SaaS dashboard shells for every category.
- Placeholder cards that say Workflow, Trust, Growth without category meaning.
- Landing pages that ignore the product category after the hero.
- Designs that ignore DESIGN.md.
- Copy that ignores strategy.md.
- Stitch prompts that attempt ten unrelated changes at once.
- Generated repos that cannot deploy to Vercel.

## Quality Gate

Before this project is considered usable, verify:

- DESIGN.md exists and is recorded in arknet.design-contract.json.
- docs/STITCH_EXPORT_BUNDLE.md exists and records whether screen/code export material was supplied.
- strategy.md exists and includes a unique core mechanic.
- docs/STITCH_REFINEMENT_PROMPTS.md includes Zoom-Out -> Zoom-In, one-change iteration, variants, and critique prompts.
- The live URL loads.
- The home page matches the product category.
- The app page performs the first useful workflow.
- Pricing explains why someone would pay.
- The generated repository contains a safe `vercel.json` without deprecated `public: true`.
- The quality report and design contract are both present.
