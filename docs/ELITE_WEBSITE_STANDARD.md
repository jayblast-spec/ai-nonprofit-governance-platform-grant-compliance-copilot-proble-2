# Elite Website Standard

This is the production bar for AI Nonprofit Governance Platform. The output is not acceptable because it deploys. It is acceptable only when it can sell, explain, demonstrate, and convert for the exact category.

## Elite Website Gate

Every generated project must pass these checks:

- Homepage depth: at least 7 meaningful sections.
- Category specificity: the homepage must match governance_platform, not a generic SaaS shell.
- Commercial structure: hero, proof, problem, mechanism, feature/use case, pricing, trust, and final conversion path.
- Working product surface: /app must demonstrate the first useful workflow.
- Pricing path: /pricing must explain why a buyer pays now, with at least three clear offers or access tiers.
- Trust/story depth: /about and /use-cases must contain real product-specific reasoning.
- Mobile polish: core pages need responsive breakpoints and no overlapping text.
- Design evidence: DESIGN.md and STITCH_EXPORT_BUNDLE.md must be present and recorded in arknet.design-contract.json.
- Copy discipline: no "generic SaaS", "powerful solution", "all-in-one platform", "Try the live workflow", or category-less CTA language.
